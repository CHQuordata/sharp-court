▎ TWO PICKS PIPELINES — shared infra, separate outputs

Per-game aiAnalyzeGame() (line 5405, ⚡ AI ANALYZE button) and batch generatePicks() (line 5145, 🤖 GENERATE button) are distinct flows that share prompt-builder helpers and learned-rules state but produce separate outputs (verdict cards vs leaderboard). Earlier versions of this diagram conflated them — corrected below.

▎ MEASURED IMPACT — what we actually verified

  • Verified ✓ — the bugs we shipped fixes for: H2H repeated 5×; cache guard always missed; prefetch ran for non-tennis; LIMITED DATA flag fired on partial gaps; cross-midnight live matches lost from TODAY; ghost games on TMW; suspect _repairJSON output landing in picks_history; XSS payload in verdict-card model output; #1 click opening a different matchup. All reproduced and fix-verified.
  • Verified ✓ — synthetic test of tournament-load logic: 2 matches at 6-3 6-4 / 7-6 4-6 6-3 → correctly counts 2m/5s/51g and emits ⚠ flag at gap≥20.
  • Not measured ⊘ — pick win-rate before vs after any prompt-quality patch (#1, #2, #4, #5, #8). No A/B run. No retroactive replay against last 30 days of grading. The "we got fucked today" Sabalenka/Paolini/Bencic upset cluster post-dates the prompt fixes — we cannot say whether the prompt changes were neutral, positive, or negative.
  • Not measured ⊘ — whether the model attends to the new Tournament load: line at all. The signal is in the prompt; downstream usage is opaque without per-pick reasoning audits.
  • Not measured ⊘ — score calibration. We capped the display at 10 (#10) but raw scores still hit 46+. We don't know what the score-to-win-rate relationship actually is. Sorting picks descending by an uncalibrated score still puts whatever the model labels "highest score" first — that label may not correlate with picks that win.
  • Honest gap ⚠ — no logging of which prompt blocks the model cited as decisive in its verdicts. Closing the loop between "we added X to the prompt" and "the model used X to make the call" requires a verdict-reasoning capture we don't currently store.

▎ OUTCOMES NOT MEASURED (read this BEFORE you ascribe wins)

  • Pick win-rate — no before/after measurement of any prompt change. The prompt is denser and more honest about gaps. Whether picks land more often is unmeasured.
  • Prompt utilization — we added Tournament load + canonical dedup to the input. We do not capture whether the model cited those blocks in its reasoning. Closing this loop needs a verdict-reasoning audit.
  • Score calibration — raw scores still hit 46. Sorting by an uncalibrated score puts the model's most-confident pick at the top, not necessarily the most-likely winner. Today's Sabalenka/Paolini/Bencic upset cluster is consistent with calibration drift on heavy-favorite clay matches.
  • Ensemble agreement — Haiku verifier output stored but not summarized in any KPI. We don't know the disagree-rate trend.

▎ OPEN WORK — round-3 priorities (newly surfaced)

  • [HIGHEST PRIORITY] Score calibration audit — the score scale is unbounded; capping display masks the issue. Needs: histogram of historical raw scores vs win-rate per bucket; either re-scale at emission, or drop "score" as a sort key in favor of EV-weighted ranking.
  • Verdict-reasoning capture — log a structured "decisive_signals" array per pick so we can tell whether new prompt blocks (tournament load, scoped flags) actually shift picks. Without this, every prompt addition is a faith-based change.
  • Tournament load: walkover/bye credit — current implementation only counts played matches. A player who got a bye is undercounted for "easy path" — should emit a positive freshness flag, not just absence of load.
  • Heavy-favorite clay regime detection — today's losses skew toward favored players on clay. Add a system-prompt rule that requires explicit clay-surface evidence (current-year clay W-L on opponent's tour) before high-confidence on -200+ ML.
  • matchstat name resolver — round-1 carryover. Diacritic / Jr. / hyphen variants still silently miss. Needs normalize-and-retry.
  • Backfill cost — 20 parallel ESPN fetches per session. Dedup by tour-week if hot path.
  • Picks-tab "TODAY'S PICKS" header — still says today even when generated 11pm and viewed 1am next morning.
  • Doubles enrichment — tournament load runs in singles branch only. Doubles bypasses by design but should at least emit a "no individual signals" note explicitly.
  • _pinnacleWorkerDown auto-recovery — round-1 carryover (M1). 2-line fix.
  • et_picks_history size cap — round-1 carryover (M7). Will matter once data accumulates.
  • Promise.race fetch failure logging — round-1 carryover (H3). Logging hygiene only.
